# 📚 Learnings: AIDA Workshop Exercise (2026-01-14)

## Context
Task: Create 60-minute AIDA copywriting workshop for food delivery professionals
Issues identified: Missing content density question, page break failures, no PDF default

---

## 🔴 Critical Failures Identified

### 1. Missing Intake Question
**Issue:** Content density question was in protocol but NOT asked
**Root Cause:** Orchestrator didn't enforce all questions from `@presentation-maker`
**Fix:** Added content density to `@orchestration-agent` first response template

### 2. Page Break Failures
**Issue:** Print output broke across pages incorrectly
**Root Cause:** Missing explicit page constraints in CSS
**Fix:** Implemented strict page CSS:
```css
@media print {
    @page {
        size: A4 portrait;  /* or landscape for slides */
        margin: 0 !important;  /* REMOVES browser headers/footers */
    }
    
    .page {
        width: 210mm !important;
        min-height: 297mm !important;
        max-height: 297mm !important;
        page-break-after: always !important;
        page-break-inside: avoid !important;
        overflow: hidden !important;
    }
    
    /* NEVER break these */
    table, .card, .exercise-box {
        page-break-inside: avoid !important;
        break-inside: avoid !important;
    }
}
```

### 3. Browser Headers/Footers in Print
**Issue:** Timestamp, URL, page numbers added by browser
**Root Cause:** `@page { margin: X }` leaves space for browser chrome
**Fix:** `@page { margin: 0 }` + content handles its own padding

### 4. Missing Default Outputs
**Issue:** PDF and Gamma-ready not generated by default
**Root Cause:** Not in default output checklist
**Fix:** Updated protocols to make these mandatory

## ✅ New Defaults for All Presentation/Visual Outputs

### Mandatory Questions
1. Content density (Executive/Balanced/Detailed)
2. Audience
3. Print format preference (A4/A5, landscape/portrait)

### Mandatory Outputs
1. **HTML** — Interactive, works in browser
2. **PDF-ready HTML** — With print button, proper page breaks
3. **Gamma-ready Markdown** — For presentations (with `[VISUAL: ...]` hints)

### Print CSS Requirements
- `@page { margin: 0 }` — Always
- Explicit page dimensions (`width: 210mm`, `height: 297mm`)
- `page-break-after: always` on page containers
- `page-break-inside: avoid` on all cards, tables, boxes
- `overflow: hidden` on pages to prevent bleed
- `-webkit-print-color-adjust: exact` for backgrounds

## 🔧 Reusable CSS Template

```css
/* === UNIVERSAL PRINT CSS (copy to all HTML outputs) === */
@media print {
    @page {
        size: A4 portrait;  /* Change to landscape for slides */
        margin: 0 !important;
    }
    
    html, body {
        margin: 0 !important;
        padding: 0 !important;
        -webkit-print-color-adjust: exact !important;
        print-color-adjust: exact !important;
    }
    
    .page, .slide {
        width: 210mm !important;      /* A4 width */
        height: 297mm !important;     /* A4 height */
        min-height: 297mm !important;
        max-height: 297mm !important;
        padding: 15mm !important;
        margin: 0 !important;
        page-break-after: always !important;
        page-break-inside: avoid !important;
        overflow: hidden !important;
        box-shadow: none !important;
        border: none !important;
    }
    
    .page:last-child, .slide:last-child {
        page-break-after: auto !important;
    }
    
    /* Never break these elements */
    table, .card, .box, .chart-container, .example-box,
    .highlight-box, .instruction-card, .writing-area {
        page-break-inside: avoid !important;
        break-inside: avoid !important;
    }
    
    /* Hide screen-only elements */
    .nav-controls, .print-btn, .sidebar, .toolbar {
        display: none !important;
    }
}
```
