# Agent Architect — Changelog

All notable changes to the Agent Architect.

---

## [2.0] — January 2026

### Added
- **Memory Protocol Integration**
  - MEMORY.md added to required files for all created agents
  - Phase 0: Memory Check — Learn from similar existing agents before creating
  - Memory Protocol section in agent template
  - Memory-informed agent design

- **Claude Skills Format**
  - YAML frontmatter added to agent template
  - Includes: name, description, version, allowed-tools
  - Compatible with Claude Code/Claude.ai standards

- **Category-Based Organization**
  - Agents now organized into categories: meta, analysis, creation, strategy, new
  - Category selection added to intake questions
  - File locations updated to `NEW/[category]/[agent-name]/`

- **Enhanced Validation Checklist**
  - Added: Memory Ready check
  - Added: Claude Skills Format check
  - Added: Category Assigned check

- **Dual-Format Catalog Updates**
  - YAML index entry for AI parsing
  - Markdown entry for human reading
  - Trigger phrases for agent discovery

- **Auto-Update Suggestions Backlog**
  - When creating agent from backlog, auto-update status to Implemented
  - Add implementation date

### Changed
- Workflow expanded from 6 phases to 7 phases (added Phase 0)
- Agent template now includes Memory Protocol section
- File output locations changed to category-based structure
- Catalog update format now dual (YAML + Markdown)

---

## [1.0] — December 2025

### Added
- Initial release
- Full framework database (business, writing, design, presentation, data, strategic)
- Expert database by domain
- 6-phase workflow (Discovery → Documentation)
- Agent structure template
- Validation checklist with expert simulation
- 3 interaction modes (Guided, Rapid, Iterative)
- Quality tiers (Essential, Standard, Comprehensive)
- Orchestration protocol and handoff format
