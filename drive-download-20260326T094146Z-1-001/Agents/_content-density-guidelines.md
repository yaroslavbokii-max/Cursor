# Content Density Guidelines

> **Version:** 2.0  
> **Created:** 2026-01-13  
> **Updated:** 2026-01-13  
> **Purpose:** Flexible content density based on user preferences and context

---

## 🎯 Core Principle

**Density is a CHOICE, not a rule.** Different contexts need different approaches.

```
Right Density = User Preference + Audience Needs + Delivery Context
```

---

## 🔴 IMPORTANT: Always Ask User Preference

Before generating content, the Orchestrator MUST ask:

```markdown
**Content Density Preference:**

A) 📊 **Executive/McKinsey** — Minimal text, maximum impact
   Best for: C-suite, board meetings, keynotes
   ~40-60 words/slide | 3-4 bullets max

B) 📋 **Balanced** — Key info with context (RECOMMENDED DEFAULT)
   Best for: Team meetings, workshops, training
   ~80-100 words/slide | 4-5 bullets

C) 📝 **Detailed/Reference** — Comprehensive, standalone
   Best for: Handouts, self-study, documentation
   ~120-150 words/slide | 5-7 bullets

D) 🎯 **Custom** — Specify your requirements
```

## 📊 Presentations / Slides — By Style

### Style A: Executive/McKinsey (960×540px slide)

| Element | Limit | Notes |
|---------|-------|-------|
| Words per slide | 40-60 | One key message |
| Bullet points | 3-4 | No sub-bullets |
| Lines per bullet | 1 | Single thought |
| Font size (body) | 18-20px | Large, readable from distance |
| Font size (headers) | 36-40px | Bold, commanding |
| Whitespace | 40% | Breathing room |

**Best for:** Board presentations, investor pitches, keynotes

### Style B: Balanced (960×540px slide)

| Element | Limit | Notes |
|---------|-------|-------|
| Words per slide | 80-100 | Key message + context |
| Bullet points | 4-5 | Occasional sub-bullets OK |
| Lines per bullet | 1-2 | Complete thoughts |
| Font size (body) | 16-18px | Readable |
| Font size (headers) | 32-36px | Clear hierarchy |
| Whitespace | 30% | Comfortable |

**Best for:** Team meetings, workshops, training sessions

### Style C: Detailed/Reference (960×540px slide)

| Element | Limit | Notes |
|---------|-------|-------|
| Words per slide | 120-150 | Comprehensive |
| Bullet points | 5-7 | Sub-bullets allowed |
| Lines per bullet | 2-3 | Full explanations |
| Font size (body) | 14-16px | Smaller but readable |
| Font size (headers) | 28-32px | Functional |
| Whitespace | 20% | Dense but organized |

**Best for:** Handouts, self-study materials, reference decks

## ✅ Validation Checklist (All Styles)

Regardless of density chosen:

- [ ] Content fits within slide/page boundaries
- [ ] No overflow into footer zone
- [ ] Readable from intended distance
- [ ] Clear visual hierarchy
- [ ] One main message per slide (even if detailed)

## 🚦 Density Levels

### Too Dense ❌

```
Slide packed with 8 bullet points, each 2-3 lines long,
with a cluttered diagram and small 10px font that's
hard to read from the back of the room.
```

**Signs:**
- No white space visible
- Font smaller than standard
- Reader feels overwhelmed
- Multiple messages competing

### Just Right ✅

```
┌─────────────────────────────────────┐
│   The MECE Principle                │
│                                     │
│   ○ Mutually Exclusive              │
│                                     │
│   ○ Collectively Exhaustive         │
│                                     │
│            [Diagram]                │
│                                     │
└─────────────────────────────────────┘
```

**Signs:**
- Clear focal point
- Obvious visual hierarchy
- Reader can grasp in 10 seconds
- Room to breathe

### Too Sparse ❌

```
┌─────────────────────────────────────┐
│                                     │
│                                     │
│              MECE                   │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

**Signs:**
- Missing context
- Wasted space
- No clear takeaway
- Audience confused

## 🔧 Quick Decision Rules

### Slide Has Too Much Content If:
- You need to shrink font below 14px
- There are more than 5 bullet points
- You can't read it in 10 seconds
- There's no visible white space

### Solution:
**Split into multiple slides.** Don't shrink.

---

### Worksheet Has Too Much Content If:
- Exercise instructions exceed 4 lines
- Answer boxes are smaller than 60px
- Tables would need to break across pages
- Font is below 11pt

### Solution:
**Add more pages.** Don't cram.

---

### Cheatsheet Has Too Much Content If:
- Text overflows the page
- Font is below 9pt
- Sections are crowded together
- No visual separation between items

### Solution:
**Use A4 instead of A5** or **add pages.**
