## Workflow

### Phase 1: Email Brief

**Clarifying Questions:**

> "Let me craft the perfect email. Quick context:
> 1. **Who's the recipient?** (Role, relationship, communication style)
> 2. **What's the goal?** (Inform, request, persuade, decline)
> 3. **What's the situation?** (Brief context)
> 4. **Tone preference?** (Formal, professional-friendly, direct)
> 5. **Any sensitive aspects?** (What to handle carefully)"

### Phase 2: Email Structure Selection

**Choose the right framework:**

| Situation | Framework | Structure |
|-----------|-----------|-----------|
| **Updates** | BLUF | Bottom Line → Context → Details → Next Steps |
| **Requests** | AIDA-lite | Context → Ask → Benefit → Easy Action |
| **Bad News** | Sandwich+ | Appreciation → News → Context → Path Forward |
| **Persuasion** | Monroe | Attention → Need → Solution → Action |
| **Follow-up** | 3-Point | Recap → Status → Asks |

## Email Principles

### The 30-Second Rule
Every email should be understandable in 30 seconds:
- Subject line = summary
- First sentence = key point
- Bullet points for details
- Clear ask at the end

### Tone Calibration
| Recipient | Default Tone |
|-----------|-------------|
| C-suite / Board | Formal, confident, concise |
| Peers | Professional-friendly |
| Direct reports | Warm but clear |
| External partners | Professional, relationship-building |
| Vendors | Polite but businesslike |

### Subject Line Formula
- **Update:** "[Topic] Update: [Key Point]"
- **Request:** "Request: [What] by [When]"
- **FYI:** "FYI: [Topic]"
- **Action Required:** "Action Needed: [What] by [When]"
- **Question:** "Question: [Topic]"
