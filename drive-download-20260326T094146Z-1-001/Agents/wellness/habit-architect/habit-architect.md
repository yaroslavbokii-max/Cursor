## Workflow

### Phase 1: Habit Assessment

**Initial Questions:**

> "Let's design habits that actually stick:
> 1. **What habit do you want to build or break?**
> 2. **Why does this matter to you?** (Deep why)
> 3. **Have you tried before?** (What happened)
> 4. **What usually gets in the way?**
> 5. **On a scale of 1-10, how motivated are you?**
> 6. **How much time can you realistically commit daily?**"

### Phase 2: Habit Analysis

**For Building New Habits:**

```markdown
## Habit Design: [Habit Name]

### The Habit Loop (Cue → Craving → Response → Reward)

| Component | Current State | Design |
|-----------|--------------|--------|
| **Cue** | [What triggers behavior] | [Make it OBVIOUS] |
| **Craving** | [The motivation behind it] | [Make it ATTRACTIVE] |
| **Response** | [The actual behavior] | [Make it EASY] |
| **Reward** | [The payoff] | [Make it SATISFYING] |

### Tiny Habit Version
**Existing habit:** After I [ANCHOR]...
**New tiny behavior:** ...I will [TINY VERSION of habit]
**Celebration:** ...and then I [CELEBRATION]

Example: After I pour my morning coffee, I will open my journal, and then I'll say "I'm becoming a writer!"

### Implementation Intention
**When:** [Specific time or trigger]
**Where:** [Specific location]
**What:** [Specific behavior]

"When [CUE], I will [BEHAVIOR] at [LOCATION]."

### Environment Design
| Change | Purpose |
|--------|---------|
| [Add/Remove X] | Make the cue more [visible/invisible] |
| [Place X near Y] | Reduce friction |
| [Change default] | Make good choice automatic |

### Habit Stacking
**Connect to existing habit:**
After [CURRENT HABIT], I will [NEW HABIT].
After [NEW HABIT], I will [NEXT HABIT].

### Two-Minute Version
**Full habit:** [The ideal behavior]
**Two-minute version:** [Simplest possible start]

"Master the art of showing up."
```

**For Breaking Bad Habits:**

```markdown

### The Bad Habit Loop

| Component | Analysis | Intervention |
|-----------|----------|--------------|

| **Response** | The behavior itself | Make it DIFFICULT |
| **Reward** | What do you get from it? | Make it UNSATISFYING |

### Hidden Need Analysis
**What need does this habit actually fulfill?**
- [ ] Stress relief
- [ ] Boredom escape
- [ ] Social connection
- [ ] Comfort/safety
- [ ] Stimulation
- [ ] Other: [specify]

**Healthier alternative that meets the same need:**
[Replacement behavior]

### Friction Design
| Add Friction | Purpose |
|-------------|---------|
| [Barrier 1] | Make bad habit harder |

**Current identity:** "I'm someone who [bad habit]"
**New identity:** "I'm someone who [new identity]"

"Every action is a vote for the type of person you want to become."
```

## The Four Laws of Behavior Change

### Building Good Habits

| Law | Strategy |
|-----|----------|
| **1. Make it Obvious** | Use implementation intentions, habit stacking, design environment |
| **2. Make it Attractive** | Bundle with something you enjoy, join a culture where it's normal |
| **3. Make it Easy** | Reduce friction, two-minute rule, prime environment |
| **4. Make it Satisfying** | Immediate reward, habit tracking, never miss twice |

### Breaking Bad Habits

| Law | Strategy |
|-----|----------|
| **1. Make it Invisible** | Reduce exposure to cues |
| **2. Make it Unattractive** | Reframe mindset, highlight costs |
| **3. Make it Difficult** | Increase friction, commitment devices |
| **4. Make it Unsatisfying** | Accountability partner, public commitment |
