## Phase 4: Visual Generation

### Technical Implementation

**For Business Users (Non-Technical):**
- Generate **self-contained HTML files** (everything embedded)
- No external dependencies (no CDN links, no external files)
- Open HTML file → See visual → Print it
- Works offline, works everywhere

**Format Generation:**
1. **Printable HTML** (self-contained, embedded SVG/CSS)
2. **SVG code** (for embedding in other documents)
3. **Print-optimized version** (CSS print media queries, page breaks)
4. **Web-ready version** (responsive, interactive-ready)

**Code Structure (with overlap prevention):**
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>[Visual Title]</title>
    <style>
        /* Embedded CSS - no external dependencies */
        /* Bolt brand colors or custom brand */
        /* Print-optimized styles */
        
        /* CRITICAL: Prevent overlapping elements */
        body {
            margin: 0;
            padding: 0.5in;
            overflow: hidden;  /* Prevents content overflow */
        }
        
        .visual-container {
            margin: 0.5in 0;
            text-align: center;
            clear: both;  /* Critical: prevents float overlap */
            overflow: hidden;  /* Prevents content overflow */
        }
        
        .visual-container svg {
            max-width: 100%;
            height: auto;
            display: block;  /* Prevents inline spacing issues */
            margin: 0 auto;  /* Centers the visual */
        }
        
        /* Prevent text overflow */
        .text-content {
            word-wrap: break-word;  /* Breaks long words */
            overflow-wrap: break-word;
        }
    </style>
</head>
<body>
    <div class="visual-container">
        <!-- Embedded SVG visual -->
        <!-- Fully rendered, ready to print -->
    </div>
</body>
</html>
```

**CRITICAL:** Always include `clear: both` and `overflow: hidden` in CSS to prevent overlapping elements.

**CRITICAL: SVG Text Alignment (MANDATORY)**

All text in SVG visuals MUST be properly centered and aligned:

```svg
<!-- ✅ CORRECT: Centered text in circle -->
<circle cx="225" cy="225" r="50" fill="#F9F9F9" stroke="#333333"/>
<text x="225" y="225" text-anchor="middle" dominant-baseline="middle" font-size="16" fill="#333333">Label</text>

<!-- ✅ CORRECT: Centered text (not in circle) -->
<text x="225" y="100" text-anchor="middle" dominant-baseline="middle" font-size="24" fill="#333333">Label</text>

<!-- ❌ WRONG: Left-aligned text (default) -->
<text x="225" y="100" font-size="24" fill="#333333">Label</text>

<!-- ❌ WRONG: Text not vertically centered in circle -->
<text x="225" y="225" text-anchor="middle" font-size="16" fill="#333333">Label</text>
```

**Requirements:**
- [ ] All SVG text uses `text-anchor="middle"` for horizontal centering
- [ ] All SVG text uses `dominant-baseline="middle"` for vertical centering
- [ ] Text coordinates (x, y) match element centers (cx, cy for circles)
- [ ] Text is positioned within visual boundaries
- [ ] Text does not overflow containers
- [ ] SVG viewBox includes all content with padding (viewBox="0 0 [width] [height]" with content within)

**SVG ViewBox Requirements:**
- [ ] ViewBox must be large enough to contain all elements
- [ ] Add padding in viewBox (e.g., if content is 400x400, use viewBox="0 0 450 450")
- [ ] No elements extend beyond viewBox boundaries

**Chart Design Requirements (MANDATORY):**
- [ ] Charts must have clear "So What?" - the key insight must be visually obvious
- [ ] Use brand accent color (e.g., Solstice Gold #FFD700) for vital few/key insights
- [ ] Use secondary color (e.g., Soft Pebble #E1E1E1) for the rest
- [ ] Charts must be MECE (clear, non-overlapping categories)
- [ ] All labels must be centered and readable

- **Process flow:** Sequence of hat usage

- **Double Diamond:** Discover, Define, Develop, Deliver

- Different audiences need different versions

1. **Layout Variations:** Different arrangements of same content

4. **Format Variations:** Horizontal, vertical, square, custom aspect ratios

## Constraints & Guardrails

### ALWAYS Do:
- ✅ Generate actual rendered visuals (not descriptions)
- ✅ Create self-contained HTML files for business users
- ✅ Auto-select best visual type based on content/context
- ✅ Provide recommendations with alternatives
- ✅ Validate quality (readability, accessibility, brand compliance)
- ✅ Generate multiple formats (HTML, SVG, print-ready)
- ✅ Apply brand guidelines (Bolt default, customizable)
- ✅ Auto-detect frameworks and use framework-specific visuals
- ✅ Offer complexity levels (simple, standard, complex)
- ✅ Include usage recommendations
- ✅ Ensure accessibility (WCAG, alt text, ARIA)
- ✅ Prevent overlapping elements (CSS: clear: both, overflow: hidden)
- ✅ Use default language: English (user can override)

### NEVER Do:
- ❌ Provide only descriptions or specifications
- ❌ Require external dependencies for HTML files
- ❌ Skip quality validation
- ❌ Ignore framework detection
- ❌ Create visuals without accessibility features
- ❌ Use dark backgrounds (unless context requires)
- ❌ Generate visuals that don't match content
- ❌ Skip brand guidelines (always apply default or custom)
