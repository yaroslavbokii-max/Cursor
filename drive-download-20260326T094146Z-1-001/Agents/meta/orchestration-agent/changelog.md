# Orchestration Agent — Changelog

All notable changes to the Orchestration Agent.

## [2.0] — January 2026

### Added
- **Memory Protocol Integration**
  - Phase 0: Memory Injection — Read agent memories before workflow
  - Phase 6: Memory Update — Capture learnings after workflow
  - Memory context in agent handoffs
  - Cross-agent learning patterns

- **Retrospective Questions (Phase 5)**
  - Overall satisfaction scoring (1-10)
  - Agent performance feedback
  - Missing capability identification
  - What worked well capture
  - User preference updates

- **Catalog-Driven Agent Discovery**
  - Trigger phrase matching against YAML index
  - Works-with combination suggestions
  - Memory-informed agent selection

- **Updated Agent Selection Matrix**
  - Added @visual-designer to presentations
  - Added @facilitation-guide-generator to workshops
  - Added @database-architect to products
  - Added @code-generator to products
  - Added @seo-optimizer to web products
  - Added @form-generator for surveys
  - Added @data-enrichment-agent for unstructured data
  - Added @document-processor for file extraction
  - Added @interactive-content-compiler for HTML
  - Added @personalized-plan-generator for plans

- **Enhanced Handoff Format**
  - Memory context section (learnings, preferences, anti-patterns)
  - Collaboration insights from past workflows

- **Claude Skills Format Compatibility**
  - YAML frontmatter for potential Claude Code/Claude.ai integration

### Changed
- Workflow expanded from 5 phases to 7 phases
- Agent plan display now includes memory insights
- Gap analysis moved to Phase 7
